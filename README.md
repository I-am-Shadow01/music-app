# Music App (Android, standalone — ไม่ผ่าน Discord, ไม่ต้องมี backend แยก)

แอปเล่นเพลง Android ที่ค้นหา + ดึงเสียงจาก YouTube **ในตัวแอปเองทั้งหมด**
ผ่านไลบรารี NewPipeExtractor (ตัวเดียวกับที่แอป NewPipe ใช้) ไม่มี backend/server
แยกให้ต้องรันเองอีกต่อไป และไม่ใช้ Lavalink/Discord เลย

## โครงสร้าง
- Kotlin + Jetpack Compose (UI)
- Media3 ExoPlayer + MediaSessionService (เล่นเพลงพื้นหลังได้)
- NewPipeExtractor (ห่อด้วย NewValve) — ค้นหา + ดึงลิงก์เสียงจาก YouTube ตรงในแอป
- แท็บ **ตั้งค่า**:
  - ธีม (ตามระบบ/สว่าง/มืด) + เลือกสีหลักได้เอง (ม่วง/น้ำเงิน/เขียว/ส้ม/แดง)
  - คุณภาพเสียง (ต่ำ–สูงสุด)
  - เปิด/ปิดเล่นเพลงถัดไปในคิวอัตโนมัติเมื่อจบเพลง
  - ล้างแคชลิงก์เสียง
  - เปิด/ปิดเช็คอัปเดตอัตโนมัติตอนเปิดแอป + ปุ่ม **"เช็คอัปเดตตอนนี้"** กดเช็คเองได้ทุกเมื่อ
  - ดูเลข build ปัจจุบัน + ลิงก์ไป GitHub Releases

## วิธี build เป็น APK (ไม่ต้องมี Android Studio)

โปรเจกต์นี้มี GitHub Actions (`.github/workflows/build.yml`) ที่ compile APK ให้อัตโนมัติทุกครั้งที่ push:

1. push โค้ดโฟลเดอร์นี้ขึ้น repo บน GitHub
2. เข้าแท็บ **Actions** ของ repo → รอ workflow "Build Debug APK" รันจบ (สีเขียว)
3. กดเข้าไปที่ run ล่าสุด → เลื่อนลงมาที่ **Artifacts** → ดาวน์โหลด `music-app-debug`
4. แตกซิปได้ไฟล์ `app-debug.apk` → โอนเข้ามือถือ → ติดตั้งได้เลย (เปิด "อนุญาตติดตั้งจากแหล่งที่ไม่รู้จัก" ถ้าเครื่องถาม)

## ใช้งาน

เปิดแอปแล้วค้นหาเพลงได้เลย ไม่ต้องตั้งค่าอะไรเพิ่ม — ไม่ต้องรัน `music-backend` หรือเปิด
port forwarding ใน Linux Terminal อีกต่อไป (ของเดิมที่เคยตั้งไว้เอาไว้ปิด/ลบทิ้งได้)

## หมายเหตุ

- **อัปเดตอัตโนมัติแล้ว (แบบเช็คก่อนค่อยทำ):** workflow `.github/workflows/update-newpipe.yml` เช็ค
  เวอร์ชันล่าสุดของ NewPipeExtractor จาก GitHub ให้เองทุกวัน 03:00 UTC ถ้าเจอเวอร์ชันใหม่กว่าที่ pin
  ไว้ใน `gradle.properties` จะ commit + push อัปเดตเลขเวอร์ชันให้เอง ซึ่งจะไป trigger
  `build.yml` ให้ build APK ใหม่ตามอัตโนมัติ — **ถ้าไม่มีเวอร์ชันใหม่ ก็จะไม่ทำอะไรเลย** ไม่ build
  ทิ้งเปล่าๆ ทุกวัน
- เช็คประวัติการอัปเดตย้อนหลังได้ที่แท็บ **Commits** ของ repo (จะเห็น commit อัตโนมัติ
  "chore: bump NewPipeExtractor to vX.X.X" ทุกครั้งที่มีการอัปเดต)
- **แอปอัปเดตตัวเองได้แล้ว:** ทุกครั้งที่ build ผ่าน CI จะสร้าง GitHub Release แนบ APK ไว้อัตโนมัติ
  (tag `build-<เลขรัน>`) ตัวแอปจะเช็คตอนเปิดแอปว่ามี release ใหม่กว่าที่ติดตั้งอยู่ไหม ถ้ามีจะขึ้นแถบ
  "มีเวอร์ชันใหม่พร้อมอัปเดต" ให้กดอัปเดตได้ในแอปเลย (ครั้งแรกอาจต้องอนุญาต "ติดตั้งแอปจากแหล่งนี้"
  ก่อน แอปจะพาไปหน้าตั้งค่าให้เอง)
- ถ้าเวอร์ชันใหม่ที่ auto-bump มาดันมีบั๊กใหม่ ให้บอกได้ จะช่วย revert commit กลับไปเวอร์ชันที่เสถียรให้
- ใช้เพื่อการศึกษา/ส่วนตัวเท่านั้น
